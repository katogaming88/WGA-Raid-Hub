Archive Current Season

Archiving pushes the current Season Name, Start Date, and End Date into the season history list. Once archived, the season appears in the Season toolbar dropdown so officers can filter by it at any time. After archiving, set a new Season Name and Start Date for the next season.

Rollover workflow:
  1. Set an End Date if the season closes before the next one starts.
  2. Click Archive Current Season below.
  3. Set the new Season Name and Start Date.
  4. Re-import this season's loot via Loot > Import -- entries are tagged with the new name automatically.
  5. Click Refresh from WCL on the Attendance tab to pull the new season's raids.

Move the current season into history so it appears in the season selector dropdown.

Officer Workflow Reference

Import RCLootCouncil loot history

  1. Confirm Season Name is set in Season Settings (e.g. "Season 2"). New entries will be tagged with this label.
  2. In-game, open RCLootCouncil and use its export feature to copy your loot history JSON.
  3. Go to Loot > Import, paste the JSON into the text box, and click Import.
  4. Duplicate entries are skipped automatically -- you can paste multiple nights at once or re-paste old exports safely.

Refresh attendance from Warcraft Logs

  1. Go to the Attendance tab.
  2. Click Refresh from WCL. This pulls the latest raid nights from Warcraft Logs and populates the attendance grid.
  3. Select a raid night from the dropdown to review or edit player statuses.

Manually edit attendance status

  1. Go to the Attendance tab and select the raid night from the dropdown.
  2. Find the player in the grid and use the status dropdown on their row to set: Present, Bench, Excused, Medical Leave, No Show, or Not on Roster.
  3. The change saves immediately -- a checkmark confirms success.

Commit attendance scores to the roster sheet

  1. After reviewing the attendance grid, click Commit Scores to Sheet on the Attendance tab.
  2. Confirm the prompt. This calculates each player's attendance percentage and writes it to the Google Sheet roster.
  3. Run this after any manual edits or a WCL refresh to keep the public roster view up to date.

Set season start date and season name

  1. Go to Season Settings.
  2. Set the Season Start Date to the first day of the current raid season. Raids before this date are excluded from attendance scoring.
  3. Set the Season Name (e.g. "Season 2"). This label is applied to all future loot imports -- set it before importing loot.

Season reset workflow

  1. Go to Season Settings and update the Season Start Date to the new season's first raid day.
  2. Update the Season Name (e.g. "Season 2") so new loot imports are tagged correctly.
  3. Go to Loot > Import History and click Clear All Loot History to remove last season's entries.
  4. Re-import this season's loot via Loot > Import.
  5. On the Attendance tab, click Refresh from WCL to pull the new season's raid nights.
  6. Commit updated scores via Commit Scores to Sheet.
